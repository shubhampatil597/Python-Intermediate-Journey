# Functions as First-Class Objects

Python treats functions as objects.

Functions can:

- Be stored in variables
- Be passed as arguments
- Be returned from functions

---

# Closures

A closure remembers variables from its outer scope even after the outer function finishes execution.

Benefits:

- State Preservation
- Cleaner Code
- Framework Internals
- Authentication Systems

---

# Decorators

Decorators enhance functions without modifying their original code.

Benefits:

- Reusability
- Logging
- Monitoring
- Validation
- Authentication

---

# Parameterized Decorators

Decorators can receive arguments.

Example:

```python
@repeat(3)
```

Useful when decorator behavior must be configurable.

---
